# Skill: research

## Context
Deep-dive information gathering with source validation. Use when the user needs facts, patterns, or competitive intelligence before any build decision.

## Instructions
1. Identify the core question — strip away framing bias
2. Search authoritative sources (S-level: official docs, GitHub, academic)
3. Cross-reference at least 2 independent sources for every claim
4. Deduplicate findings — if 3 sources say the same thing, cite the most authoritative
5. Return: summary + sources + confidence level (high/medium/low)

## Constraints
- Never cite without a URL or source identifier
- Never present speculation as fact
- If sources conflict, surface the conflict explicitly

## Examples
**Input:** "What are the best free proxies for web scraping?"
**Output:** Table comparing Webshare (1GB), Oxylabs (5GB), ProxyScrape (unlimited but unreliable) + risk assessment + recommendation tree
